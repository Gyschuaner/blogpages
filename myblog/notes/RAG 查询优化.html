    RAG 查询优化

    作者 2024-01-01 大约 15 分钟 RAG 检索优化 查询分解 大语言模型

    查询优化必要性

    在RAG（检索增强生成）系统中，查询优化是提高检索质量的关键环节。传统的单一查询方法可能无法捕获复杂问题的多个方面，导致检索结果不够全面。通过多种查询优化策略，可以显著提升检索的召回率和精确率，从而为后续的生成提供更优质的上下文信息。

    多角度查询(Multi-Query)

    核心思想

    多角度查询的核心思想是将用户的一个原始查询转换为多个不同角度的表述，然后分别进行检索，最后合并结果。这样可以从不同的视角捕获与问题相关的信息，提高检索的全面性。

    基础代码实现

    from langchain.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

def generate_query_variants(original_query: str, llm: ChatOpenAI) -> list[str]:
    """生成查询变体"""
    prompt = ChatPromptTemplate.from_messages([
        ("system", "你是一个专业的查询改写助手。请将用户的查询转换为3个不同角度的表述，以捕获查询的不同方面。每个表述应该保持查询的核心意图，但使用不同的措辞和视角。每个变体应该在新的一行开始。"),
        ("human", "原始查询: {query}")
    ])
    
    chain = prompt | llm | StrOutputParser()
    result = chain.invoke({"query": original_query})
    variants = [variant.strip() for variant in result.split("\n") if variant.strip()]
    
    # 确保返回3个变体，如果不足则补充
    while len(variants) < 3:
        variants.append(original_query)
    
    # 添加原始查询
    variants = [original_query] + variants
    
    return variants[:4]  # 返回原始查询 + 最多3个变体

# 使用示例
llm = ChatOpenAI(temperature=0)
original_query = "RAG系统中的查询优化技术有哪些？"
query_variants = generate_query_variants(original_query, llm)
print("查询变体:")
for i, variant in enumerate(query_variants, 1):
    print(f"{i}. {variant}")

    优缺点分析

    优点：

    • 实现简单，易于集成到现有系统
    • 可以从多个角度捕获查询意图
    • 对于模糊或复杂的查询特别有效

    缺点：

    • 需要额外的LLM调用开销
    • 变体质量依赖于提示词设计
    • 结果合并可能引入噪声

    融合式检索(RAG-Fusion)

    核心思想

    RAG-Fusion是一种高级检索策略，它结合了多角度查询和结果融合技术。通过生成多个查询变体，分别检索后使用特殊的融合算法（如倒数排序融合）来合并结果，从而提高检索质量。

    倒数排序融合算法

    倒数排序融合（Reciprocal Rank Fusion）是一种有效的结果融合方法，它考虑了文档在不同查询结果中的相对位置。

    Python实现

    from typing import List, Dict, Any
from langchain.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI


def reciprocal_rank_fusion(results_lists: List[List[Dict]], k: int = 60) -> List[Dict]:
    """
    实现倒数排序融合算法
    
    Args:
        results_lists: 多个查询结果列表
        k: 融合参数
        
    Returns:
        融合后的文档列表
    """
    # 存储每个文档的融合得分
    fused_scores = {}
    
    # 对每个查询结果列表
    for results in results_lists:
        # 对每个文档赋予基于其排名的得分
        for rank, doc in enumerate(results, 1):
            doc_id = doc.get("id", str(hash(doc.get("page_content", ""))))
            # 倒数排序融合公式
            score = 1 / (k + rank)
            if doc_id in fused_scores:
                fused_scores[doc_id]["score"] += score
            else:
                fused_scores[doc_id] = {"doc": doc, "score": score}
    
    # 按得分排序并返回
    sorted_docs = sorted(fused_scores.values(), key=lambda x: x["score"], reverse=True)
    return [item["doc"] for item in sorted_docs]


class RAGFusion:
    """RAG-Fusion 实现"""
    
    def __init__(self, retriever, llm):
        self.retriever = retriever
        self.llm = llm
    
    def generate_queries(self, original_query: str, num_queries: int = 4) -> List[str]:
        """生成多个查询变体"""
        prompt = ChatPromptTemplate.from_messages([
            ("system", f"你是一个专业的查询改写助手。请将用户的查询转换为{num_queries-1}个不同角度的表述，以捕获查询的不同方面。每个表述应该保持查询的核心意图，但使用不同的措辞和视角。每个变体应该在新的一行开始。"),
            ("human", "原始查询: {query}")
        ])
        
        chain = prompt | self.llm | StrOutputParser()
        result = chain.invoke({"query": original_query})
        variants = [variant.strip() for variant in result.split("\n") if variant.strip()]
        
        # 添加原始查询并确保数量足够
        all_queries = [original_query] + variants
        return all_queries[:num_queries]
    
    def retrieve_and_fuse(self, queries: List[str], k: int = 4) -> List[Dict]:
        """执行多个查询并融合结果"""
        # 为每个查询获取结果
        all_results = []
        for query in queries:
            results = self.retriever.get_relevant_documents(query, k=k)
            # 转换为字典格式（如果需要）
            result_dicts = []
            for i, doc in enumerate(results):
                # 确保是字典格式
                if hasattr(doc, "dict"):
                    result_dicts.append(doc.dict())
                elif isinstance(doc, dict):
                    result_dicts.append(doc)
                else:
                    # 如果是其他类型，尝试转换
                    result_dicts.append({"page_content": str(doc), "id": str(i)})
            all_results.append(result_dicts)
        
        # 融合结果
        fused_results = reciprocal_rank_fusion(all_results)
        return fused_results
    
    def query(self, original_query: str, num_queries: int = 4, k: int = 4) -> List[Dict]:
        """执行RAG-Fusion查询"""
        # 生成多个查询
        queries = self.generate_queries(original_query, num_queries)
        
        # 检索并融合
        results = self.retrieve_and_fuse(queries, k)
        
        return results

    代码详解

    reciprocal_rank_fusion函数

    • 参数：
      • results_lists：多个查询结果列表
      • k：融合参数，默认60
    • 返回值：融合后的文档列表
    • 核心逻辑：对每个查询结果中的文档，根据其排名计算倒数分数，并累加所有查询中同一文档的分数，最后按分数排序返回

    RAGFusion类

    • 初始化参数：
      • retriever：文档检索器
      • llm：大语言模型实例
    • generate_queries方法：使用LLM生成多个查询变体
    • retrieve_and_fuse方法：执行多个查询并融合结果
    • query方法：整合查询生成和结果融合的完整流程

    查询分解(Query Decomposition)

    递归分解策略

    查询分解是将复杂问题拆解为多个简单子问题，分别解答后再综合的方法。递归分解策略在处理每个子问题时，会利用之前子问题的答案作为上下文，逐步构建对复杂问题的理解。

    Python实现

    from typing import List, Dict, Any
from langchain.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI


class RecursiveDecomposition:
    """递归查询分解"""
    
    def __init__(self, vectorstore, llm):
        self.vectorstore = vectorstore
        self.retriever = vectorstore.as_retriever()
        self.llm = llm
    
    def decompose_query(self, question: str) -> List[str]:
        """分解复杂查询为子问题"""
        decompose_prompt = ChatPromptTemplate.from_messages([
            ("system", """将复杂问题分解为2-4个简单的子问题。
            要求：
            1. 子问题应该按逻辑顺序排列
            2. 每个子问题都应该是独立可回答的
            3. 每行一个问题"""),
            ("human", "{question}")
        ])
        
        chain = decompose_prompt | self.llm | StrOutputParser()
        sub_questions_str = chain.invoke({"question": question})
        sub_questions = [q.strip() for q in sub_questions_str.split("\n") if q.strip()]
        
        return sub_questions
    
    def answer_sub_question(
        self, 
        question: str, 
        context: str = ""
    ) -> str:
        """回答单个子问题"""
        # 检索相关文档
        docs = self.retriever.get_relevant_documents(question)
        doc_context = "\n\n".join([doc.page_content for doc in docs[:3]])
        
        # 构建提示词
        if context:
            full_context = f"已知信息:\n{context}\n\n相关文档:\n{doc_context}"
        else:
            full_context = f"相关文档:\n{doc_context}"
        
        answer_prompt = ChatPromptTemplate.from_messages([
            ("system", "基于提供的信息简洁地回答问题。"),
            ("human", "{context}\n\n问题: {question}")
        ])
        
        chain = answer_prompt | self.llm | StrOutputParser()
        answer = chain.invoke({
            "context": full_context,
            "question": question
        })
        
        return answer
    
    def query(self, question: str) -> dict:
        """执行递归分解查询"""
        print(f"📋 原始问题: {question}\n")
        
        # 1. 分解问题
        sub_questions = self.decompose_query(question)
        print(f"🔍 分解为 {len(sub_questions)} 个子问题:")
        for i, sq in enumerate(sub_questions, 1):
            print(f"   {i}. {sq}")
        
        # 2. 递归回答
        accumulated_context = ""
        sub_answers = []
        
        print("\n💡 逐步回答:")
        for i, sq in enumerate(sub_questions, 1):
            print(f"\n   子问题{i}: {sq}")
            answer = self.answer_sub_question(sq, accumulated_context)
            sub_answers.append(answer)
            accumulated_context += f"\n\n问题{i}: {sq}\n答案: {answer}"
            print(f"   答案: {answer[:100]}...")
        
        # 3. 综合最终答案
        final_prompt = ChatPromptTemplate.from_messages([
            ("system", "基于以下子问题和答案，综合回答原始问题。"),
            ("human", """原始问题: {question}
            
			子问题和答案:
			{sub_qa}

			请给出完整的综合答案:""")
        ])
        
        sub_qa = "\n\n".join([
            f"Q{i}: {q}\nA{i}: {a}" 
            for i, (q, a) in enumerate(zip(sub_questions, sub_answers), 1)
        ])
        
        chain = final_prompt | self.llm | StrOutputParser()
        final_answer = chain.invoke({
            "question": question,
            "sub_qa": sub_qa
        })
        
        return {
            "question": question,
            "sub_questions": sub_questions,
            "sub_answers": sub_answers,
            "final_answer": final_answer
        }

        docs = await self.retriever.aget_relevant_documents(question)
        doc_context = "\n\n".join([doc.page_content for doc in docs[:3]])
        
        answer_prompt = ChatPromptTemplate.from_messages([
            ("system", "基于文档简洁回答问题。"),
            ("human", "文档:\n{context}\n\n问题: {question}")
        ])
        
        chain = answer_prompt | self.llm | StrOutputParser()
        answer = await chain.ainvoke({
            "context": doc_context,
            "question": question
        })
        
        return answer
    
    async def query_async(self, question: str) -> dict:
        """异步执行并行分解"""
        # 1. 分解问题
        sub_questions = self.decompose_query(question)
        
        # 2. 并行回答所有子问题
        tasks = [
            self.answer_sub_question_async(sq) 
            for sq in sub_questions
        ]
        sub_answers = await asyncio.gather(*tasks)
        
        # 3. 综合答案（同递归版本）
        # ...
        
        return {
            "question": question,
            "sub_questions": sub_questions,
            "sub_answers": sub_answers,
            "final_answer": final_answer
        }

# 使用
parallel_decomp = ParallelDecomposition(vectorstore, llm)
result = asyncio.run(
    parallel_decomp.query_async("Python vs JavaScript in Web Development")
)

    这段代码用到了很多异步执行：

    • async def answer_sub_question_async(self, question: str) -> str:：这是一个协程函数 (Coroutine Function) 的声明（使用 async def）。它是一个可以被暂停和恢复的特殊函数。当它被调用时，它返回一个协程对象 (Coroutine Object)，这个对象必须被'驱动'（如被 await）才能执行。
      • docs = await self.retriever.aget_relevant_documents(question)：这是一个await 表达式。它用于暂停当前协程 (answer_sub_question_async) 的执行，直到 self.retriever.aget_relevant_documents() 这个可等待对象 (Awaitable) 完成。aget... 是 get... 的异步版本。在等待期间，事件循环 (Event Loop) 会切换去执行其他已就绪的协程。
      • answer = await chain.ainvoke(...)：这是第二次 await。协程再次暂停，等待 chain.ainvoke (LLM链的异步调用) 完成。这展示了 asyncio 的核心优势：在一个协程的生命周期中，可以多次在I/O密集型操作（如数据库查询、API调用）上'让出'控制权。
    • async def query_async(self, question: str) -> dict:：这是主协程函数，它负责编排整个并行工作流。
      • tasks = [ self.answer_sub_question_async(sq) for sq in sub_questions ]：这是一个列表推导式 (List Comprehension)。它不会执行协程，而是创建了一个协程对象 (Coroutine Object) 的列表。此时，这些协程尚未开始执行；它们只是被"实例化"了，准备被调度。
      • sub_answers = await asyncio.gather(*tasks)：这是并发执行 (Concurrent Execution) 的核心。asyncio.gather() 函数接收一个或多个可等待对象 (*tasks 使用"解包"操作符将 tasks 列表中的所有协程对象作为独立参数传入)。gather 会并发地调度所有这些协程在事件循环上运行。await 在这里会暂停 query_async，直到 tasks 列表中的所有协程都执行完毕。sub_answers 将是一个列表，其顺序与 tasks 列表的顺序一一对应。
    • asyncio.run( parallel_decomp.query_async(...) )：这是启动异步事件循环的入口点。asyncio.run() 函数负责创建和管理事件循环，运行传入的主协程 (query_async(...)) 直到它完成，然后关闭事件循环。这是在顶层（非 async 函数中）启动异步代码的标准方式。
