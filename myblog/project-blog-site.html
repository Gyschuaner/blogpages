Web / Blog

个人博客基地

一个 MC 风格的个人主页与博客站点，用来沉淀 AI、代码、学习笔记和之后真正完成的项目。

项目目标

把个人主页、博客、时间线、技能和项目展示统一成一个有辨识度的小站。视觉上保持方块、箱子、像素边框这些 MC 元素，但阅读内容要比纯装饰更清爽。

已完成

  • 搭建了首页、博客页、项目页、时间线等基础页面。
  • 博客页支持按分类箱子筛选文章，并接入打开/关闭箱子图标。
  • 文章正文页统一成更适合阅读的 MC 风格排版。
  • 搭建了 obsidian-blog-publisher skill，把 Obsidian 笔记发布成博客文章的流程标准化。
  • 通过 GitHub 仓库连接 Cloudflare Pages 自动部署。

Skill 发布流水线

这个博客不是手动复制 HTML 维护，而是把“写 Markdown - 生成文章 - 更新列表 - 构建验证 - 发布上线”封装成了一个可复用的 Codex skill。之后只需要在 Obsidian 里写好 Markdown，给出文件路径，skill 就会按同一套规则把它变成站内文章。

  • 读取 Obsidian Markdown，识别 frontmatter、标题、标签、代码块、表格、数学公式、Wiki 链接和图片引用。
  • 把笔记转换成统一的文章页，保留 MC 风格导航、正文排版、目录、代码高亮和图片懒加载。
  • 自动更新博客注册表，让新文章进入博客列表、分类筛选、搜索和侧边栏。
  • 构建前后执行语法检查、静态构建和差异检查，避免样式断裂、路径错误或坏脚本上线。
  • 发布时走 GitHub 到 Cloudflare Pages 的部署链路，形成从本地知识库到公网博客的闭环。

能力亮点

这个 skill 最有价值的地方不是“生成一篇网页”，而是把重复工作沉淀成稳定接口：同样的输入格式、同样的样式约束、同样的验证步骤。它让博客更像一个小型内容工程系统，也展示了把个人工作流产品化、自动化和可维护化的能力。
