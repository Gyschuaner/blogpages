## 三种最具代表性的经典范式

**ReAct**：Reasoning and Acting，将“思考”和“行动”紧密结合的范式，让智能体边想边做，动态调整。
**Plan-and-Solve**：三思而后行，智能体首先生成一个完整的行动计划，然后严格执行。
**Reflection**：自我批判和修正，通过自我批判和修正来优化结果，提升答案质量。

## 封装基础 LLM 调用函数

为了让代码结构更清晰、更易于复用，我们来定义一个专属的LLM客户端类。这个类将封装所有与模型服务交互的细节，让我们的主逻辑可以更专注于智能体的构建。

```python
import os  
from openai import OpenAI  
from dotenv import load_dotenv  
from typing import List, Dict  
  
# 加载.env文件中的环境变量  
load_dotenv()  
  
class HelloAgentsLLM:  
    def __init__(self, model: str = None, apikey: str = None, base_url: str = None, timeout: int = None):  
        self.model = model or os.getenv("LLM_MODEL_ID")  
        self.apiKey = apikey or os.getenv("LLM_API_KEY")  
        self.base_url = base_url or os.getenv("LLM_BASE_URL")  
        self.timeout = timeout or int(os.getenv("LLM_TIMEOUT", "60"))  
  
        if not all([self.model, self.apiKey, self.base_url]):  
            raise ValueError("模型ID、API密钥和基础URL必须提供")  
  
        self.client = OpenAI(  
            api_key=self.apiKey,  
            base_url=self.base_url,  
            timeout=self.timeout  
        )  
  
    def think(self, messages: List[Dict[str, str]], temperature: float = 0):  
        # 调用LLM思考，返回响应内容  
        print(f"正在调用{self.model}模型...")  
        try:  
            response = self.client.chat.completions.create(  
                model=self.model,  
                messages=messages,  
                temperature=temperature,  
                stream=True, # 启用流式响应  
            )  
  
            # 处理流式响应  
            print("大模型响应成功：")  
            collected_content = []  
            for chunk in response:  
                content = chunk.choices[0].delta.content or ""  
                print(content, end="", flush=True)  # 实时打印内容  
                collected_content.append(content)  
            print()  # 换行  
            return "".join(collected_content)  
  
        except Exception as e:  
            error_msg = f"调用LLM API时发生错误,错误信息: {str(e)}"  
            print(error_msg)  
            return None  
  
#--- 客户端使用示例 ---
if __name__ == "__main__":  
    try:  
        llmClient = HelloAgentsLLM()  
        example_messages = [  
            {"role": "system", "content": "你是一个友好的助手。"},  
            {"role": "user", "content": "你好，今天的天气怎么样？"}  
        ]  
        print("调用LLM")  
        response = llmClient.think(example_messages)  
        if response:  
            # print("\n\nLLM响应内容：", response)  
            print("LLM调用成功！")  
    except ValueError as e:  
        print(f"初始化LLM客户端失败: {str(e)}")
```

```python
    def think(self, messages: List[Dict[str, str]], temperature: float = 0):

                print(content, end="", flush=True)  # 实时打印内容  
                collected_content.append(content)  
            print()  # 换行  
            return "".join(collected_content)

            print(error_msg)  
            return None  
```
*  这一段最大的特点就是启用了流式响应`stream=True`
	* 当启用了流式响应之后，模型会一段字一段字的生成，像流水一样，和现在市面上的多数大模型一样。
	* 没有启动流式响应，模型就会一次性全部生成所有的文字，这样会导致等待时间更长。
* `think(self, messages: List[Dict[str, str]], temperature: float = 0)`:
	* `messages: List[Dict[str, str]]`代表了`message`是一个列表`List`，里面存放了字典`Dict`，而字典的键值对都是`str`字符串。
	* `temperature`代表创意度，0代表最严谨。
* `for chunk in response(...)`：
	* `chunk`指开启流式生成后的一段段文字小块
	* `content = chunk.choices[0].delta.content or ""`
		* 取`choices[0]`是因为chunk中可能有好几个备选答案，我们只取第一个。
		* `.delta`是“增量”的意思，代表只取新增的部分，而不是取所有已经生成的内容。
		* `or ""`是一个保险，万一这一个`chunk`是空的，则用一个空字符串代替。
	* `flush=True`表示让电脑实时打印内容
	* `return "".join(collected_content)`表示将`collected_content`列表收集到的所有`chunk`合并起来，形成一句完整的话。

## ReAct

ReAct由Shunyu Yao于2022年提出，其核心思想是模仿人类解决问题的方式，将**推理 (Reasoning)** 与**行动 (Acting)** 显式地结合起来，形成一个"思考-行动-观察"的循环。![[Pasted image 20251103214105.png]]

### Plan-and-Solve

这种范式将任务处理明确地分为两个阶段：**先规划 (Plan)，后执行 (Solve)**。

如果说 ReAct 像一个经验丰富的侦探，根据现场的蛛丝马迹（Observation）一步步推理，随时调整自己的调查方向；那么 Plan-and-Solve 则更像一位建筑师，在动工之前必须先绘制出完整的蓝图（Plan），然后严格按照蓝图来施工（Solve）。事实上我们现在用的很多大模型工具的Agent模式都融入了这种设计模式。

Plan-and-Solve Prompting 由 Lei Wang 在2023年提出。其核心动机是为了解决思维链在处理多步骤、复杂问题时容易"偏离轨道"的问题。

### Reflection

Reflection 机制的核心思想，正是为智能体引入一种**事后（post-hoc）的自我校正循环**，使其能够像人类一样，审视自己的工作，发现不足，并进行迭代优化。

Reflection 机制的灵感来源于人类的学习过程：我们完成初稿后会进行校对，解出数学题后会进行验算。这一思想在多个研究中得到了体现，例如 Shinn, Noah 在2023年提出的 Reflexion 框架。其核心工作流程可以概括为一个简洁的三步循环：**执行 -> 反思 -> 优化**。
